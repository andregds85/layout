Docker

Docker é uma ferramenta que cria micro maquinas virtuais com a finalidade de não "sujar" o ambiente do desenvolvedor

Dessa forma você pode ter dois ambientes configurados sem um conflitar um com Outro

Imagem

É como se fosse uma iso do Sistema Operacional

Serve para se tornar uma maquina virtual e se tornar um container

As imagens podem ser limpas como as que você usa como as que você usa para formatar um computador ou customizalas com vários aplicativos instalados por exemplo Apache, mysql, jekins, nodejs e etc ...

Qualquer coisa que pode ser feita em linha de comando pode servir de configuração para a imagem

Container

É uma maquina virtual depois de pronta para uso iniciada apartir de uma imagem . Trata-se de um ambiente completo que pode ser acessado via linha de comando. Containers Ligam e Desligam instantaneamente e podem ser "resetados" ao desligar ou manter as congurações para a próxima vez que forem ligados.

Pensem assim Você baixa uma imagem iso, formata seu computador instala uns programas e pronto ele é um container

O poder e a Velocidade do docker vem da vantagem de que todas as imagens já estão prontas para uso e não há perda de tempo para formatação da máquina.

Então se você rodar uma imagem do MYSQL o docker vai fazer o download e sem nenhuma interação o servidor ja vai estar

Manter o Container Rodando

$ docker run ubumtu bash

Este Comando vai montar toda configuração e te dar acesso ao shell root do container.

Equanto você não digitar exit e der enter o container permanecerá rodando

Listando Container

Cada vez que você executa o comando run, o docker vai criar um novo container do zero e armazenar em cache para utilização futura.

Você não executar o comando Run toda hora.

$ docker ps -a

O Docker cria automaticamente um nome aleatorio para cada container criado apartir do do comando run, Através desse nome você pode continuar utilizando o container sem perder os dados a cada inicialização

Para Container / Rodando Container ja Criado

$ docker start NOME_CONTAINER

Ate o comando stop ser rodado. Portanto você não tera acesso ao shell sem utilizar outro comando

Acessar comandos docker que já estão rodando

$ docker attach NOME_CONTAINER

Renomear Containers

O docker se encarrega de dar nomes aleatórios para cada container porem eles não são nada faceis de lembrar ou você acha que adoring_ardignheli servidor-teste é fácil de escrever e lembrar

o Código segue o padrão

$ docker rename NOME_PADRAO NOME_FACILDELEMBRAR

Para rodar

$ docker start servidor-teste

$ docker attach servidor-teste

Para Container

$ docker stop servidor-teste
